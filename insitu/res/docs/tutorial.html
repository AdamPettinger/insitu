Welcome to InSitu!

InSitu organizes task views using a hierarchy of Modes, Views, and Filters. Modes serve as analogues for different operating modes, such as navigation, manipulation, or survey, and are displayed in a tab bar at the top of the application. Modes serve as containers for Views, which display a ROS image stream, such as the feed from an end effector camera. Views can be augmented with useful information by adding Filters, which can range from a simple heartbeat indicator to a full on AR overlay. For example, a typical rover setup might have a navigation Mode containing front and rear camera Views with base odometry and LiDAR Filters.

To create a mode, select Edit->New Mode from the menu bar or press Ctrl+Shift+M to bring up the mode creation dialog.

To create a view, ensure there is an active mode and select Edit->New View from the menu bar or press Ctrl+Shift+V to bring up the view creation dialog.

To create a filter, ensure there is an active view and select Edit->New Filter from the menu bar or press Ctrl+Shift+F to bring up the view creation dialog.

Please refer to the project Github for more information: https://github.com/PaperFanz/insitu

You can refer back to this tutorial at any time by creating a new mode named "Tutorial."
